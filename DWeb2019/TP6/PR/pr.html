Abstract

Este trabalho consiste na construção de um servidor que suporta CRUD, direcionado a arquivos sonoros.

Para tal ustilizamos um ficheiro .json para ser o nosso ficheiro de dados: cancoes.json .

Este ficheiro foi preenchido com os dados de arq-son-EVO.xml . Como este ficheiro não tem id's, a função definida em adicionar.js que utiliza o nanoid

As páginas HTML são geradas a partir de ficheiros pug.
